Funktionsübersicht

Jedes Werkzeug, das ein Indie-Entwickler braucht

HeightForge wurde speziell für die Workflows der Spieleentwicklung entwickelt — nicht für die akademische Kartografie. Jede Funktion existiert, weil ein Entwickler sie brauchte.

Kern-Workflow

5-Schritte-Terrain-Pipeline

Ein geführter Workflow, der die komplexe Terrain-Erstellung in klare, handhabbare Schritte strukturiert. Keine Angst mehr vor dem leeren Canvas.

01
Island Mask
Küste definieren
Male eine grobe Inselform oder generiere sie automatisch mit Slidern für Küstenbreite, Polygonform und Rauheit. Diese Maske bestimmt, wo Land existiert.
02
Forest + Mountains
Terrain-Typen schichten
Wälder sammeln sich in Tälern, Berge erheben sich auf Kämmen. Jeder Layer wird organisch mit Domain-Warp-Filtern für natürlich wirkende Übergänge gemischt.
03
Rivers (Fluid sim)
Wasserfluss simulieren
Die gradientenbasierte Fluidsimulation gräbt Flussbetten entlang des Weges des geringsten Widerstands. Steuere Länge und Verzweigung mit einem einzigen Slider.
04
Erosion Pro
Erosions-Nodes anwenden
Verbinde hydraulische, thermische und Wind-Erosions-Nodes in einer visuellen Pipeline. Jeder Node ist GPU-beschleunigt und läuft in einem Hintergrund-Thread.
05
Export

Systemanforderungen

HeightForge Studio läuft auf Windows, macOS und Linux. Du benötigst Python 3.11 oder neuer und pip.

  • Python 3.11+ (3.12 empfohlen)
  • Mindestens 4 GB RAM, 8 GB empfohlen für 4K+
  • Beliebige GPU (optional) — Der CPU-Modus funktioniert für die meisten Aufgaben gut

Hinweis: HeightForge wird für die Beta als Quellcode verteilt. Eine Standalone-Binary (kein Python erforderlich) ist für v1.0 geplant.

Installation

1. Abhängigkeiten installieren

Entpacke das heruntergeladene Archiv und führe aus:

# Kern-Abhängigkeiten (erforderlich) pip install numpy opencv-python PySide6 opensimplex # Optional: 3D-Viewport pip install pyqtgraph PyOpenGL # Optional: EXR-Export (Unreal Engine) pip install openexr

2. Anwendung starten

python heightforge/main.py

3. GPU-Beschleunigung (optional)

Installiere CuPy für NVIDIA-Karten. Die App erkennt dies automatisch und fällt lautlos auf die CPU zurück, falls nicht verfügbar.

# NVIDIA CUDA 12.x pip install cupy-cuda12x # AMD ROCm pip install cupy-rocm

HeightForge testet beim Start eine echte CUDA-Kernel-Kompilierung. Wenn die GPU fehlschlägt, fällt es ohne Absturz auf die CPU zurück. Keine Konfiguration nötig.

Dein erstes Terrain

Startbildschirm

Wähle deine Arbeitsauflösung. 2048×2048 wird für die meisten Projekte empfohlen — es lässt sich schnell bearbeiten und kann beim Export hochskaliert werden. Wähle ein Biom-Preset oder starte mit einem leeren Projekt.

Schritt 1 — Insel-Maske

Klicke auf Automatisch Generieren mit den Standardeinstellungen. Du erhältst eine prozedurale Inselform. Passe Küste und Rauheit an und generiere neu oder male manuell mit den Pinsel-Werkzeugen.

  • Malen — fügt der Maske hinzu
  • Radieren — entfernt aus der Maske
  • Weichen — glättet harte Kanten

Schritte 2–4

Arbeite Wald, Berge und Flüsse nacheinander ab. Jeder Layer wird durch die Insel maskiert, sodass nichts in den Ozean überläuft. Nutze Strg+Z jederzeit — der Undo-Verlauf ist 20 Schritte tief.

Erosion Pro

Klicke auf den ⚗ Erosion Pro-Button, um den Node-Editor zu öffnen. Füge Nodes aus der Palette hinzu und ziehe Verbindungen zwischen den In- und Output-Sockets. Führe die Pipeline aus — sie läuft in einem Hintergrund-Thread, damit die UI reaktionsfähig bleibt.

Tipp: Starte mit Hydraulic Erosion → Thermal Erosion → Normalize. Dies ist die klassische geologische Sequenz und liefert schnell realistische Ergebnisse.

In deine Engine exportieren

Unity (PNG 16-bit)

Nutze Datei → Heightmap exportieren, wähle PNG 16-bit in deiner Zielauflösung. In Unity funktioniert Terrain → Import Raw nicht mit PNG — nutze stattdessen ein Skript oder den Import Heightmap-Button des Terrain Tools-Packages.

Unreal Engine (EXR oder PNG)

Exportiere als EXR 32-bit für maximale Präzision. In Unreal: Landscape Editor → Import → EXR-Datei auswählen. Stelle sicher, dass der Height Type auf Float32 eingestellt ist.

Godot (PNG 16-bit)

Exportiere als PNG 16-bit. In Godot: Erstelle eine HeightMapShape3D und nutze die map_data-Eigenschaft mit einem Skript, um das PNG über Image.load() zu laden.

Kachel-Export für sehr große Karten

Nutze für Auflösungen über 8K den Tiled Export im Export-Schritt. Er rendert in 2048px-Kacheln, um Speicherüberläufe zu vermeiden. Die Ausgabe ist immer RAW 16-bit Binär.

# Import RAW in Unity per Script using UnityEngine; using System.IO; byte[] raw = File.ReadAllBytes("heightmap_8192.raw"); // 16-bit little-endian, zu 0–1 normalisieren für TerrainData
